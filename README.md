# Credit Card Fraud Detection - Exploratory Data Analysis

## Background
This project is an Exploratory Data Analysis (EDA) on a credit card 
transaction dataset to identify patterns and characteristics of fraudulent 
transactions. This analysis is supported by my experience as an IT Monitoring 
staff at a payment switching company, where understanding real-time transaction 
anomalies is part of daily operations.

## Objectives
- Understand the distribution and patterns of credit card transaction data
- Identify features most related to fraudulent transactions
- Provide data-driven insights and business recommendations

## Dataset
- Source: Kaggle - Credit Card Fraud Detection (ULB)
- Total transactions: 284,807
- Fraud transactions: 492 (0.17%)
- Features: Time, V1-V28 (PCA result), Amount, Class
